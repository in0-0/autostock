서비스 개요

AutoStock은 대한민국 주식시장(KOSPI/KOSDAQ)을 대상으로 하는 주말 배치형 투자 분석 MVP다. 현재 구현은 자동 주문 서비스가 아니라, 포트폴리오 상태와 시장/재무/기술 조건을 평가해 로컬 JSON 산출물과 Telegram용 Markdown 리포트 캐시를 생성하는 CLI 파이프라인이다.

기본 실행 명령은 python3 -m src.main --settings config/settings.yaml이며, 기본 설정은 외부 증권사 API 없이 MockBrokerConnector와 샘플 시장 데이터를 사용한다.

원자적 파일 저장 portfolio state, explain log, report JSON을 atomic write로 저장한다. 테스트 검증

현재 생성되는 산출물

  • data/portfolio_state.json: 병합된 포트폴리오 상태, 예수금, 평가금액, 포지션 비중, 실패 소스와 경고.
  • data/explain_logs/explain_YYYY-MM-DD.json: 매크로 상태, 필터별 판단, 최종 순위, provider, 리스크 사유.
  • data/reports/report_YYYY-MM-DD.json: Telegram Markdown 본문 캐시.
  • CLI 표준 출력: 실행 시 생성된 Markdown 리포트를 콘솔에 출력.

현재 상태에서 확인할 수 있는 것

확인 가능 항목 근거 한계
샘플 설정으로 CLI가 끝까지 실행되는지 python3 -m src.main --settings config/settings.yaml smoke run 통과. 샘플 데이터 기반이므로 실제 시장/계좌 상태와 무관하다.
핵심 회귀 동작 python3 -m pytest 기준 38개 테스트 통과. 테스트는 단일 테스트 파일 중심이며, 외부 네트워크 호출은 mock/fixture로 대체된다.
Google Sheets 파서와 readonly 호출 계약 한국어 헤더, 숫자 파싱, 민감값 비노출 경고, fake service의 values.get 호출 테스트가 있다. 실제 Google API 인증, 권한, 네트워크, 시트 공유 상태는 이 환경에서 확인되지 않았다.
pykrx/FDR 데이터가 들어왔을 때 내부 기술 시계열로 바뀌는 방식 일봉 row를 캘린더 주/월 bucket으로 정규화하는 단위 테스트가 있다. 실제 pykrx/FDR 패키지 설치, 호출 성공, 응답 스키마 변화 대응은 로컬에서 확인되지 않았다.
데이터 불완전성에 대한 보수적 차단 macro unavailable은 CAUTION 컨텍스트와 감점으로, provider warning, missing current price, short/stale series는 후보 리스크 또는 후보 차단으로 반영되는 테스트가 있다. 실제 운영 데이터 품질 분포는 아직 관측되지 않았다.
민감 정보 노출 방지 일부 provider error redaction 테스트와 로컬 secret scan에서 민감 패턴 미검출. 모든 운영 로그/외부 서비스/사용자 입력 조합을 포괄하는 보안 검증은 아니다.
문서상 릴리스 상태 docs/STATUS.md, docs/ROADMAP.md, docs/HISTORY.md 확인. 정식 릴리스 태그는 없고 HISTORY.md에는 아직 release history가 없다.

운영 관점의 현재 결론

이 저장소는 현재 “로컬에서 반복 실행 가능한 투자 분석 MVP” 수준까지 구현되어 있다. 샘플/fixture 기반 분석, 리포트 생성, explain log, conservative blocking, CAUTION 감점, 스프레드시트 파서, provider fallback 구조, Telegram 상태 기록은 확인 가능하다.

반면 “실제 계좌와 실제 시장 데이터를 사용하는 주말 운영 서비스”라고 보기에는 아직 미확인 영역이 남아 있다. 특히 실제 브로커 커넥터, Google Sheets live 인증, pykrx/FDR live 호출, macro/fundamental live source, Telegram 발송, launchd 스케줄은 별도 운영 검증이 필요하다.

권장 다음 단계

  1. v0.2 release gate로 Google Sheets fixture + market fixture CLI 경로를 유지하고, 실제 Google readonly smoke test를 별도 로컬 설정으로 실행한다.
  2. pykrx/FDR 의존성이 설치된 환경에서 bounded universe real mode를 실행해 provider telemetry, cache, fundamental 차단과 macro CAUTION 컨텍스트 동작을 확인한다.
  3. macro/fundamental live source를 도입할지 제품 결정 후, 도입한다면 freshness와 provenance를 포함한 fixture-first 테스트를 추가한다.
  4. v0.3에서 launchd, Telegram delivery, credential split, 운영 runbook을 묶어 운영성 검증을 진행한다.
