학습 가이드 소개

이 가이드는 AutoStock의 Python 백엔드 배치 파이프라인 설계에서 파생된 소프트웨어 공학 지식을 정리한 것입니다. 단순한 코딩을 넘어 "네트워크 불안정성 속에서 결함 없는 데이터 가계도를 유지하는 배치 시스템"을 설계하기 위한 깊이 있는 레슨을 담고 있습니다.

🎯 학습 목표

1. 다형성을 극대화하여 테스트 가능성(Mockability)이 높은 파이프라인 설계 패턴 습득.
2. 원격 금융 API 호출 시 레이트 리밋과 장애를 안전하게 극복하는 복원력(Resiliency) 설계.

Core Concept

시스템의 신뢰성은 방어적 코드의 조합이다

AutoStock의 아키텍처는 네트워크 실패, 크레덴셜 누락, 그리고 이종 가격 데이터 불일치(Disagreement) 등 모든 가능한 예외 상황을 기획 단계의 "상태 사상"으로 간주하고 이를 품질 게이트(Quality Gate)로 걸러냅니다. 결코 오염된 상태가 파이프라인 핵심 비즈니스 로직(랭킹 및 연산)에 전달되지 않도록 원천 격리하는 것이 이 시스템의 핵심 철학입니다.

단방향 파이프라인 아키텍처

배치 및 데이터 수집 아키텍처에서 가장 피해야 할 것은 '상태의 스파게티화'와 '부수 효과(Side Effect)'입니다. AutoStock은 데이터 유입 단계부터 가공 및 분석 결과 기록 단계까지 엄격한 단방향 데이터 파이프라인을 고수합니다.

1. 파이프라인 파티셔닝 (Pipeline Partitioning)

시스템을 크게 세 개의 레이어로 파티셔닝하여 책임을 격리합니다.

  • Ingestion Layer: 포트폴리오(Google Sheets) 및 유니버스(KOSPI/KOSDAQ)의 원시 수집 수행.
  • Processing Layer: 가격 불일치 게이트, 재무 데이터 노멀라이저, 완성도 검사.
  • Calculation & Output Layer: 평가 엔진 작동, 포트폴리오 랭킹 부여, 리포팅 및 원자적 쓰기.
Python (src/main.py 구조적 흐름)
# 1. Ingestion
portfolio = merge_portfolio_sources(_build_portfolio_source(settings).fetch())
market_universe = _resolve_market_universe(settings)

# 2. Processing with Fallback & Gate check
market_data = load_market_data_with_fallback(build_market_data_providers(...))
_apply_price_cross_check(..., market_data)  # Disagreement gate check

# 3. Calculation & Output

⚠️ 현업 설계 팁: 원자적 쓰기 (Atomic Writes)

배치 프로세스에서 파일 기록 도중 예외가 발생하거나 프로세스가 중단되면, 파일이 불완전하게 잘려나가는(Truncation) 파일 오염이 발생합니다. AutoStock은 임시 파일(e.g., .tmp)로 먼저 전체를 완전히 쓴 다음, 성공 시 OS 레벨의 원자적 함수인 os.replace를 호출하여 기존 파일을 덮어씁니다.

Pattern 3

체인 오브 리스폰시빌리티 (Chain of Responsibility)

여러 개의 원격 소스 또는 Fallback 체인을 순회하며, 성공하는 최초의 수집기에서 데이터를 취득하고 오류는 리스트에 차곡차곡 누적하는 책임을 전가하는 패턴입니다.

네트워킹 및 회복력 설계

모든 네트워크 API(예: 한국거래소, OpenDART)는 장애가 날 수 있고, 속도 제한(Rate Limit)이 존재합니다. 프로덕션 급 파이프라인이 가져야 할 네트워킹 전략을 알아봅니다.

1. API Rate Limiting 대응 (Request Delay)

원격 금융 서버는 단기간 내 많은 요청이 올 경우 IP 차단이나 HTTP 429 Too Many Requests를 반환합니다. 수집 파이프라인 루프 내부에는 설정으로 주입 가능한 지연 시간(request_delay_seconds)을 슬립 함수와 바인딩해야 합니다.

Python
def _sleep_between_provider_requests(delay_seconds: float) -> None:
    if delay_seconds > 0:
        time.sleep(delay_seconds)

2. 데이터 소스 간 정합성 대조 (Cross-check Gate)

동일한 정보를 가리키지만 제공처가 다를 때(예: pykrx 종가 vs FDR 종가), 상호 대조(Cross-Check) 로직을 돌려 값의 괴리율이 일정 범위를 초과하면 예외 경고를 수집하여 랭킹 및 추천 엔진에 제공되지 않도록 퀄리티 게이트를 작동시킵니다.

보안 및 데이터 퀄리티 게이트

현업 환경에서 가장 민감한 정보 유출 방지 및 잘못된 알고리즘 계산을 방지하기 위한 안전장치들입니다.

구분 설계 원칙 실제 구현 기법
크레덴셜 보호 로컬 설정 파일과 환경 변수에 저장된 토큰/비밀번호가 콘솔이나 로그, 커밋 파일에 절대로 유출되지 않아야 함. redaction.py 내 정규식 기반의 sanitize_error_message를 설계하여 API Key 형태나 패스워드 패턴을 [redacted]로 치환.
신선도 검사 (Freshness) 수집된 가격 및 재무 데이터가 지나치게 과거 시점의 데이터일 경우 연산에 배제해야 함. price_max_age_days를 설정으로 두고 최종 거래일과 배치 실행일 사이의 간격이 임계치를 초과할 때 stale_price_data 에러 수집.
추적 가능성 (Lineage) 완성된 보고서 내 각각의 스크리닝 필드 수집 출처가 어디인지 규정할 수 있어야 디버깅이 가능함. FundamentalRecord 모델 내에 field_provenance(필드별 기원 정보) 사전을 영구 동반 저장.
Python (src/utils/redaction.py)
import re

def sanitize_error_message(exc: Exception) -> str:
    message = str(exc)
    # 1. API key/Token 정규식 치환
    message = re.sub(r"(key|token|auth|password)=[a-zA-Z0-9_\-]+", r"\1=[redacted]", message, flags=re.IGNORECASE)
    # 2. URL 경로 축약 마스킹
    message = re.sub(r"https?://[^\s]+", "[url]", message)
    return message
